# Datasets

Datasets are **named definitions of ClickHouse tables** that include your schema, measures, dimensions, and business logic in one place. Think of them as your semantic layer in TypeScript.

## Overview

A dataset is the single source of truth for how your application queries a table. Define it once, and reuse it across:

- **APIs** - Serve datasets as typed HTTP endpoints
- **Jobs** - Reference the same metrics in background workers
- **Dashboards** - Query datasets from your React components
- **AI Agents** - Let agents query with consistent business logic

## Quick Example

```typescript
import { dataset, field } from '@hypequery/clickhouse';

export const Users = dataset('users', {
  source: 'users',
  tenantKey: 'tenant_id',
  timeKey: 'created_at',
  fields: {
    id: field.string(),
    email: field.string(),
    status: field.enum(['active', 'churned']),
    plan: field.enum(['free', 'pro', 'enterprise']),
  },
});

// Now query it anywhere
const activeUsers = await Users.query()
  .where('status', 'eq', 'active')
  .execute();
```

## Why Datasets?

**Without datasets**, your metrics logic lives in multiple places:

- SQL in your dashboard tool
- Query builders in your API routes
- Business logic duplicated in background jobs
- Different definitions across teams

**With datasets**, you define once and reuse everywhere:

```typescript
// Define the dataset once
export const Orders = dataset('orders', {
  source: 'orders',
  tenantKey: 'tenant_id',
  timeKey: 'created_at',
  fields: {
    id: field.string(),
    amount: field.number(),
    status: field.enum(['paid', 'refunded']),
  },
});

// Use it in your API
const revenue = await Orders.query()
  .where('status', 'eq', 'paid')
  .sum('amount')
  .execute();

// Use it in a background job
const dailyRevenue = await Orders.query()
  .where('created_at', 'gte', startOfDay)
  .sum('amount')
  .execute();

// Serve it as an HTTP endpoint
serve({ revenue: Orders });
```

## Key Features

### Type Safety

Every field is typed. Autocomplete works everywhere. Refactor the dataset, and every consumer breaks loudly if incompatible.

### Multi-Tenancy Built In

Declare a `tenantKey` on the dataset. Every query inherits the tenant filter automatically.

```typescript
export const Orders = dataset('orders', {
  source: 'orders',
  tenantKey: 'tenant_id', // ← Inherited by all queries
  fields: {
    // ...
  },
});
```

### Time-Series Awareness

Declare a `timeKey` and every query knows how to filter by date ranges efficiently.

```typescript
export const Events = dataset('events', {
  source: 'events',
  timeKey: 'timestamp', // ← Enables time-based queries
  fields: {
    // ...
  },
});
```
