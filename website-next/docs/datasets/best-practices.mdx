# Best Practices

## Define datasets at the module level

<CodeBlock>
```typescript
// datasets/orders.ts
export const Orders = dataset('orders', {
  source: 'orders',
  tenantKey: 'tenant_id',
  timeKey: 'created_at',
  dimensions: { /* ... */ },
  measures: { /* ... */ },
});
```
</CodeBlock>

## Use semantic field names

Map physical column names to semantic field names.

<CodeBlock>
```typescript
dimensions: {
  userId: dimension.string({ column: 'user_id' }),
  createdAt: dimension.timestamp({ column: 'created_at' }),
  isActive: dimension.boolean({ column: 'is_active' }),
}
```
</CodeBlock>

## Create filtered measures for common patterns

<CodeBlock>
```typescript
measures: {
  completedRevenue: measure.sum('amount', {
    filters: [eq('status', 'completed')],
  }),
  refundedRevenue: measure.sum('amount', {
    filters: [eq('status', 'refunded')],
  }),
}
```
</CodeBlock>

## Define metrics for KPIs

<CodeBlock>
```typescript
const revenue = Orders.metric('revenue', { measure: 'revenue' });
const orderCount = Orders.metric('orderCount', { measure: 'orderCount' });
const aov = Orders.metric('averageOrderValue', {
  uses: { revenue, orderCount },
  formula: ({ revenue, orderCount }) => divide(revenue, nullIfZero(orderCount)),
});
```
</CodeBlock>

## Use runtime tenancy

Declare `tenantKey` and provide tenant context at runtime.

<CodeBlock>
```typescript
await executor.metric(revenue, {}, {
  runtime: {
    tenant: { id: req.user.tenantId },
  },
});
```
</CodeBlock>

## Avoid non-public APIs

These are intentionally not part of the public API:

- `dataset.query(...)`
- root exports for dataset endpoint execution helpers
- deep imports from package internals
- automatic relationship JOIN execution
- cross-dataset derived metrics

For package integration, Hypequery packages use `@hypequery/datasets/internal`. Do not import it in application code.
