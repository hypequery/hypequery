# Frequently Asked Questions

## General

<Accordions type="single">
  <Accordion title="What is hypequery?">
    hypequery is a code-first analytics layer for ClickHouse. It gives TypeScript teams a typed query builder, reusable query definitions, and an optional serve runtime for HTTP, docs, and framework handlers.
  </Accordion>

  <Accordion title="Is hypequery production-ready?">
    Yes. hypequery is used in production environments for typed, reusable analytics queries.
  </Accordion>

  <Accordion title="What databases does hypequery support?">
    Currently ClickHouse only.
  </Accordion>

  <Accordion title="Is hypequery open source?">
    Yes. hypequery is open source and available on GitHub.
  </Accordion>
</Accordions>

## Usage

<Accordions type="single">
  <Accordion title="Can I use raw SQL?">
    Yes. Use `raw()` for SQL fragments or `rawQuery()` when you need a full raw SQL query.
  </Accordion>

  <Accordion title="How do I handle mutations (INSERT/UPDATE/DELETE)?">
    hypequery is optimized for analytics workloads. For mutations, use `rawQuery()`.
  </Accordion>

  <Accordion title="Does hypequery support ClickHouse materialized views?">
    Yes. Treat materialized views as regular tables in your generated schema.
  </Accordion>

  <Accordion title="How do I handle NULL values?">
    Use `isNull` and `isNotNull` operators in the query builder.
  </Accordion>
</Accordions>

## React Hooks

<Accordions type="single">
  <Accordion title="Do I need React hooks?">
    No. `@hypequery/react` is optional and only needed for React applications.
  </Accordion>

  <Accordion title="Can I use React hooks without serve?">
    No. React hooks require the serve runtime because they target HTTP routes generated from your API surface.
  </Accordion>

  <Accordion title="Do React hooks work with Next.js?">
    Yes. See the [Next.js guide](/docs/nextjs).
  </Accordion>
</Accordions>

## Performance & Caching

<Accordions type="single">
  <Accordion title="Does hypequery support caching?">
    Yes. hypequery supports query-result caching at the query builder layer with strategies such as `cache-first`, `network-first`, and `stale-while-revalidate`.
  </Accordion>

  <Accordion title="Can I stream large result sets?">
    Yes. Use `stream()` or `streamForEach()`.
  </Accordion>
</Accordions>

## Deployment

<Accordions type="single">
  <Accordion title="Where can I deploy hypequery?">
    Anywhere Node.js runs, including serverless platforms, containers, traditional servers, and compatible edge runtimes.
  </Accordion>

  <Accordion title="Do I need to run the hypequery dev server in production?">
    No. `hypequery dev` is only for local development. In production, use `api.run()` for in-process execution or mount `api.handler` / call `api.start()` for HTTP delivery.
  </Accordion>
</Accordions>
