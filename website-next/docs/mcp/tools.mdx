# MCP Tools

The MCP server exposes a small set of tools over your dataset registry. Agents do not receive a SQL tool.

## `list_datasets`

Lists available datasets.

<CodeBlock>
```json
{
  "name": "list_datasets"
}
```
</CodeBlock>

Example response:

<CodeBlock>
```json
{
  "datasets": [
    {
      "name": "orders",
      "description": "Customer orders and revenue data",
      "dimensionCount": 5,
      "metricCount": 2
    }
  ],
  "total": 1
}
```
</CodeBlock>

## `get_dataset_schema`

Returns dimensions, metrics, relationships, tenant key, and time key metadata for one dataset.

<CodeBlock>
```json
{
  "name": "get_dataset_schema",
  "arguments": {
    "dataset": "orders"
  }
}
```
</CodeBlock>

## `query_dataset`

Runs an ad hoc semantic query with dimensions, metrics, filters, time grain, ordering, and limit.

<CodeBlock>
```json
{
  "name": "query_dataset",
  "arguments": {
    "dataset": "orders",
    "dimensions": ["region", "status"],
    "metrics": ["revenue", "orderCount"],
    "filters": [
      { "field": "status", "operator": "eq", "value": "completed" }
    ],
    "grain": "month",
    "limit": 100
  }
}
```
</CodeBlock>

## `query_metric`

Runs a named metric handle attached to a dataset.

<CodeBlock>
```json
{
  "name": "query_metric",
  "arguments": {
    "dataset": "orders",
    "metric": "revenue",
    "dimensions": ["region"],
    "filters": [
      { "field": "status", "operator": "eq", "value": "completed" }
    ],
    "orderBy": [
      { "field": "revenue", "direction": "desc" }
    ],
    "limit": 10
  }
}
```
</CodeBlock>

## Prompt

The server also exposes a `dataset_guide` prompt that helps an MCP client explain how to query the registered datasets.

## Operators and grains

Supported filter operators:

- `eq`
- `neq`
- `gt`
- `gte`
- `lt`
- `lte`
- `in`
- `notIn`
- `between`
- `like`

Supported time grains:

- `day`
- `week`
- `month`
- `quarter`
- `year`
