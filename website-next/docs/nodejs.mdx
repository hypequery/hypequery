# Node.js

This guide assumes you already have:

- a typed `db` client
- an `analytics/queries.ts` file exporting `api`
- at least one routed query

If not, start with [Manual Installation](/docs/manual-installation) or [Quick Start](/docs/quick-start).

<Callout type="info">
  We've simplified the serve API in v0.2. If you're looking for the older builder-first API, see <a href="/docs/migrate-legacy-serve-api">Migrate from Legacy Serve API</a> and <a href="/docs/legacy-serve/query-definitions">Legacy Serve API</a>.
</Callout>

<Steps>
<Step>

### Install the HTTP runtime

<CodeBlock>
```bash
npm install hono @hono/node-server dotenv
npm install -D tsx typescript
```
</CodeBlock>

</Step>

<Step>

### Mount hypequery inside Hono

Create `src/app.ts`:

<CodeBlock>
```typescript
import { Hono } from 'hono';
import { api } from '../analytics/queries.js';
import { createFetchHandler } from '@hypequery/serve';

const hypequery = createFetchHandler(api.handler);
export const app = new Hono();

app.get('/', (c) => c.json({ status: 'ok' }));
app.all('/api/analytics/*', (c) => hypequery(c.req.raw));
```
</CodeBlock>

</Step>

<Step>

### Start the server

Create `src/index.ts`:

<CodeBlock>
```typescript
import 'dotenv/config';
import { serve } from '@hono/node-server';
import { app } from './app.js';

serve({
  fetch: app.fetch,
  port: Number(process.env.PORT ?? 3000),
});
```
</CodeBlock>

</Step>
</Steps>

Use [Fetch Runtime Integration](/docs/fetch) if you want the same pattern in Workers, Bun, or other Fetch runtimes.
