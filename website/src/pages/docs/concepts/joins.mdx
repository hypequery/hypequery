# Join Relationships

HypeQuery provides powerful features for managing table relationships and performing joins. This guide covers how to define and use join relationships effectively.

## Basic Joins

You can perform joins directly in your queries:

```typescript
const results = await db
  .table('orders')
  .innerJoin('users', 'user_id', 'users.id')
  .select(['orders.id', 'users.name', 'orders.amount'])
  .execute();
```

## Join Types

HypeQuery supports all standard SQL join types:

```typescript
// Inner Join
db.innerJoin('users', 'user_id', 'users.id')

// Left Join
db.leftJoin('users', 'user_id', 'users.id')

// Right Join
db.rightJoin('users', 'user_id', 'users.id')

// Full Join
db.fullJoin('users', 'user_id', 'users.id')
```

## Defining Reusable Relationships

Define relationships once and reuse them across your application:

```typescript
// Define relationships
QueryBuilder.setJoinRelationships({
  userOrders: {
    from: 'users',
    to: 'orders',
    leftColumn: 'id',
    rightColumn: 'user_id'
  },
  orderItems: {
    from: 'orders',
    to: 'order_items',
    leftColumn: 'id',
    rightColumn: 'order_id'
  }
});

// Use relationships in queries
const results = await db
  .table('users')
  .withRelation('userOrders')
  .withRelation('orderItems')
  .select(['users.name', 'orders.id', 'order_items.product_name'])
  .execute();
```

## Join Chains

Create chains of joins to traverse multiple relationships:

```typescript
QueryBuilder.setJoinRelationships({
  orderChain: [
    {
      from: 'users',
      to: 'orders',
      leftColumn: 'id',
      rightColumn: 'user_id',
      type: 'INNER'
    },
    {
      from: 'orders',
      to: 'order_items',
      leftColumn: 'id',
      rightColumn: 'order_id',
      type: 'LEFT'
    }
  ]
});

// Use the chain
const results = await db
  .table('users')
  .withRelation('orderChain')
  .select(['users.name', 'orders.id', 'order_items.product_name'])
  .execute();
```

## Table Aliases

Use aliases to join the same table multiple times:

```typescript
const results = await db
  .table('orders')
  .innerJoin('users', 'created_by', 'users.id', 'creator')
  .innerJoin('users', 'updated_by', 'users.id', 'updater')
  .select([
    'orders.id',
    'creator.name as created_by_name',
    'updater.name as updated_by_name'
  ])
  .execute();
```

## Type Safety

Join relationships are fully type-safe:

```typescript
interface Schema {
  users: {
    id: 'Int32';
    name: 'String';
  };
  orders: {
    id: 'Int32';
    user_id: 'Int32';
    amount: 'Float64';
  };
}

const db = createQueryBuilder<Schema>();

// TypeScript will catch these errors:
db.table('users').innerJoin('invalid_table', 'id', 'invalid.id'); // Error: invalid table
db.table('users').innerJoin('orders', 'invalid_column', 'orders.id'); // Error: invalid column
```
